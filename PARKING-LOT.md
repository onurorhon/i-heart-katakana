## Snapshots & Gallery

**Concept:** A "snapshot" feature presented alongside the existing "like" functionality. While liking a word saves just the word (for later practice), a snapshot saves the *composition* ŌĆō the word with the currently selected theme and font ŌĆō as a "card."

**User flow:**
- User sees a word displayed with a theme and font they love.
- They tap a camera icon to save that moment as a snapshot.
- Snapshots are collected in a "gallery" view.
- User can revisit the gallery to see their curated collection of styled cards.

**Open questions:**
- What can users do with snapshots beyond viewing? (Share as image? Use as flashcards? Set as widget?)
- Can the same word be saved multiple times with different styles?
- How is the gallery organized? (Chronological? By word? By theme?)
- Is there a limit on snapshots?

**Why parked:** Needs design exploration to define purpose and interaction model. Likes provide immediate value with lower complexity; snapshots can be layered in post-v1.
